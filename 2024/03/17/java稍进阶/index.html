File 类：

File 类代表文件或目录的路径名抽象表示。它主要用于查询文件属性、创建或删除文件、以及获取文件信息等操作。以下是一些常见的 File 类方法：

  • exists()：检查文件或目录是否存在。
  • isFile() 和 isDirectory()：判断当前对象是否代表文件或目录。
  • createNewFile()：创建新文件。
  • mkdir() 和 mkdirs()：创建新目录。
  • delete()：删除文件或目录。
  • getName()、getPath()、getAbsolutePath()：获取文件名、路径等信息。

I/O 流：

I/O 流是用于在 Java 程序中进行输入和输出操作的机制。它主要分为字节流和字符流两种类型：

  • 字节流：以字节为单位进行读写操作，适用于处理二进制数据。
    • InputStream 和 OutputStream 是字节输入流和输出流的父类。
  • 字符流：以字符为单位进行读写操作，适用于处理文本数据。
    • Reader 和 Writer 是字符输入流和输出流的父类。

常见的 I/O 流类有 FileInputStream、FileOutputStream、FileReader、FileWriter 等，用于从文件中读取数据或向文件中写入数据。

反射

反射是指在运行时检查或修改类或对象的能力。通过反射，我们可以动态地获取类的信息、调用类的方法、访问类的属性等，而无需在编译时确定这些操作。

反射提供了以下一些重要的类和接口来实现这些功能：

  1. Class 类：Class 类代表一个类或接口，在运行时，Java 虚拟机为每个加载的类维护一个对应的 Class 对象，通过该对象可以获取类的信息。
  2. Field 类：Field 类表示类的成员变量，可以用于获取和设置对象的字段值。
  3. Method 类：Method 类表示类的方法，可以用于调用类的方法。
  4. Constructor 类：Constructor 类表示类的构造方法，可以用于实例化对象。

下面介绍一下类似魔术方法的方法

  1. private void readObject(ObjectInputStream in) throws IOException, ClassNotFoundException：如果一个类实现了这个方法，那么在对象进行反序列化时，readObject() 方法会被调用，允许你手动控制对象的反序列化过程。
  2. private void readObjectNoData() throws ObjectStreamException：如果一个类实现了这个方法，那么在对象进行反序列化时，如果没有可用的默认数据，readObjectNoData() 方法会被调用。

**transient **

在 Java 中，transient 是一个关键字，用于修饰类的成员变量。当一个成员变量被声明为 transient 时，它将被标记为瞬态的，即在对象的序列化过程中会被忽略，不会被保存到输出流中。
