数据类型

1， 变量没有类型不需要声明 ， 但必须赋值

2，number(数字)，string(字符串)，tuple(元组)。这些为不可变数据，list(列表)，dictionary(字典)，set(集合)，这些为可变数据

3，列表中的元素是可以直接改变

4，元组，和列表类似，但是元组中的元素不能改变

5，字典表示一种对应关系，不可重复且不可变

类

当我们无法用单一的一个量或者函数表达一个数据时，通过类的方式，把数据的特性组合在一起，称为一类，类里面含有对象和方法（函数）

面向对象

相比较函数，面向对象是更大的封装，根据职责在一个对象中封装多个方法

1在完成某一个需求前，首先确定职责即要做的事情。

2.根据职责确定不同的对象，在对象内部封装不同的方法

3.最后完成的代码就是顺序的让不同的对象调用不同的方法

正则

目的是用更简便快捷的方式完成查找和抓取

？可以存在零次或一次

*可以存在零至无数次

+可以存在一次至无数次

{m，n} 可以存在m次到n次

<.+>贪婪匹配“任意字符”

<.+?>懒惰匹配“任意字符”

HTTP 协议是现代 Web 应用的基础协议

协议的请求-响应模型是其最基本的特征。客户端向服务器发送一个请求，服务器接收请求并返回一个响应。

报文头中包含了请求或响应的一些元信息，比如请求方法、URL、协议版本、状态码等。报文体则包含了具体的数据内容。

….

域名是互联网上用于标识和定位网站的字符串。每个网站都有一个唯一的域名，可以通过这个域名来访问网站。

….

在互联网中，每个设备都有一个唯一的 IP 地址，用于标识和定位这个设备。

IP 地址的作用是让不同的设备能够互相通信。当一个设备要向另一个设备发送数据时，它会将数据打包成 IP 数据报的格式，并在其中包含目标设备的 IP 地址和自己的 IP 地址等信息。随后，这个 IP 数据报会被传递给网络中的路由器，路由器会根据目标 IP 地址将数据报转发到下一个路由器，直到最终到达目标设备。

….

Get/Post是HTTP 协议中的两种请求方法，用于向服务器发送请求并获取响应。

Get 请求是一种非常常见的请求方法，它通常用于获取资源和查询数据。当客户端向服务器发送一个 Get 请求时，请求参数会附在 URL 后面

Post请求文件等，更为安全。 Post 请求没有长度限制，可以传输大量的数据。

浏览器先发送header，服务器会响应100，浏览器再次发送data，服务器返回200，请求成功

用户post需要用到工具

TCP/IP

是网络通信模型

TCP/IP提供了点对点链接的机制，将资料应该如何封装、寻址、传输、路由以及在目的地如何接收，都加以标准化。它将软件通信过程抽象化为四个抽象层，采取协议堆栈的方式，分别实现出不同通信协议。协议族下的各种协议，依其功能不同，分别归属到这四个层次结构之中，常视为是简化的七层OSI模型
